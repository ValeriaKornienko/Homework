Написать приложение, которое представляет собой игра "лотерея" (аналог БИНГО)

1. Программа должна случайным образом генерировать последовательность чисел от 1 до 90
2. Числа не должны повторяться
3. Программа должна позволять регистрировать произвольное количество игроков от 1 до 5
4. Каждому игроку должны выдаваться по три карточки с числами
5. По ходу розыгрыша и выпадания чисел на "барабане" проверяются карточки игроков и в случае
если все номера с карточки игрока выпали, то считается что он выиграл

---------------------------------------------------------------------------------

Классы:
1. Данные (entity) - эти классы являются "ядром" нашей структуры всех данных
2. Хранилище набора данных (repository) - как правило классы в которых создаются массивы наших основных наборов данных
3. Сервисы (services) - в них располагается вся бизнес-логика нашего приложения
4. Сервисы-утилиты (utils) - как правило, это локальные вспомогательные классы, содержащие методы для решения небольших задач
5. UI (user interface) - часть приложения для общения с пользователем (в дальнейшем мы от него откажемся)

-------------------------------------------------------------

Класс Cart:
- содержит набор из 15 случайных чисел от 1 до 90
Таких карт у нас будет количество игроков * 3

Класс Player:
- Идентификатор (nickname)
- Массив из трех элементов типа Cart


Класс LottoGame:
- подготовительные действия
    * подготовить "барабан" - сгенерировать массив из 90 элементов в котором случайным образом
      будут располагаться числа от 1 до 90 (без повторений)
    * создать карточки 15 штук ( и проверить что среди них нет "дубликатов" - сделаем позднее )

- зарегистрировать пользователей:
    * получить количество игроков
    * создаем столько экземпляров класса Player, сколько игроков было зарегистрировано
    * получить их nickname для каждого и присвоить эти значения в переменные экземпляров класса
    * "раздать" по три карточки каждому игроку

- провести розыгрыш:
    * пройти по "барабану", то есть извлечь по очереди числа из массива до тех пор пока кто-то из игроков не "выиграет"
    * при извлечении очередного числа их массива надо проверить все карточки у игроков на наличие выигрышной комбинации

- итог розыгрыша:
    * вывести победителя
